# Phase 4: CI/CD — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up GitHub Actions workflows for daily scheduled builds (with YouTube data fetching), SFTP deployment to production, and GitHub Pages deployment for preview.

**Architecture:** Two workflows: `deploy-live.yml` runs daily at 6am UTC on `main` (fetches data, commits trend history, builds, deploys via SFTP) and `deploy-preview.yml` runs on push to `develop` (fetches data, builds, deploys to GitHub Pages).

**Tech Stack:** GitHub Actions, SFTP, GitHub Pages

**Prerequisites:** Phases 1-3 must be complete. The repository must be pushed to GitHub with the required secrets configured.

### Task 1: Create production deployment workflow

**Files:**
- Create: `.github/workflows/deploy-live.yml`

- [ ] **Step 1: Write the workflow**

`.github/workflows/deploy-live.yml`:
```yaml
name: Deploy Live

on:
  schedule:
    - cron: '0 6 * * *'
  workflow_dispatch:

permissions:
  contents: write

jobs:
  deploy-live:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Fetch YouTube data and compute trends
        run: npm run build:data
        env:
          BERTA_YOUTUBE_API_KEY: ${{ secrets.BERTA_YOUTUBE_API_KEY }}

      - name: Commit updated trend history
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add data/trend-history.json
          git diff --staged --quiet || git commit -m "update trend history [skip ci]"
          git push

      - name: Build app
        run: npm run build

      - name: Deploy via SFTP
        uses: Creepios/sftp-action@v1.0.3
        with:
          host: access800891440.webspace-data.io
          port: 22
          username: ${{ secrets.FTP_USERNAME }}
          password: ${{ secrets.FTP_PASSWORD }}
          localPath: ./dist/
          remotePath: ./
```

Note: The output directory changed from `build/` (CRA) to `dist/` (Vite).

- [ ] **Step 2: Commit**

```bash
git add .github/workflows/deploy-live.yml
git commit -m "add production deployment workflow with daily schedule"
```

### Task 2: Create preview deployment workflow

**Files:**
- Create: `.github/workflows/deploy-preview.yml`

- [ ] **Step 1: Write the workflow**

`.github/workflows/deploy-preview.yml`:
```yaml
name: Deploy Preview

on:
  push:
    branches:
      - develop
  workflow_dispatch:

permissions:
  contents: write

jobs:
  deploy-preview:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Fetch YouTube data and compute trends
        run: npm run build:data
        env:
          BERTA_YOUTUBE_API_KEY: ${{ secrets.BERTA_YOUTUBE_API_KEY }}

      - name: Build app
        run: npm run build -- --base=/berta-berlin-website/

      - name: Deploy to GitHub Pages
        uses: s0/git-publish-subdir-action@develop
        env:
          REPO: self
          BRANCH: preview
          FOLDER: dist
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          MESSAGE: "Build: ({sha}) {msg}"
```

Note: Vite uses `--base` flag instead of CRA's `PUBLIC_URL` env var. The output folder is `dist/` not `build/`.

- [ ] **Step 2: Commit**

```bash
git add .github/workflows/deploy-preview.yml
git commit -m "add preview deployment workflow for develop branch"
```

### Task 3: Configure GitHub repository secrets

This is a manual step that cannot be automated in the workflow.

- [ ] **Step 1: Document required secrets**

The following secrets must be configured in the GitHub repository settings (Settings > Secrets and variables > Actions):

1. **`BERTA_YOUTUBE_API_KEY`** — YouTube Data API v3 key (from berta-bot's `.env` file)
2. **`FTP_USERNAME`** — SFTP username for `access800891440.webspace-data.io` (already configured in old repo)
3. **`FTP_PASSWORD`** — SFTP password (already configured in old repo)

`GITHUB_TOKEN` is automatically provided by GitHub Actions.

- [ ] **Step 2: Verify secrets are set**

After pushing the repo to GitHub, check that all secrets are configured by navigating to the repository Settings > Secrets and variables > Actions.
